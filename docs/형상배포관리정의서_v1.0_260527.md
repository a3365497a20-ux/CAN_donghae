## 1. 소스 버전 관리 전략

### 1.1 현행 방식

본 프로젝트는 공유 폴더 기반 파일 관리를 기본으로 하며, Git 도입을 권장한다.

| 구분 | 현행 | 권장 (향후) |
|------|------|------------|
| 버전 관리 | 공유 폴더 + 수동 파일 관리 | Git 로컬 저장소 |
| 브랜치 전략 | 단일 폴더 | main / feature/{기능명} |
| 병합 방식 | 파일 덮어쓰기 | Pull Request → Merge |
| 이력 추적 | 파일명에 날짜 포함 (예: search_v2.jsp) | Git commit history |

### 1.2 Git 도입 시 권장 브랜치 전략

```
main
 └── feature/auth          (인증 모듈)
 └── feature/asset-search  (자산 검색)
 └── feature/reservation   (예약 모듈)
 └── feature/navigation    (길찾기)
 └── feature/admin         (관리자 기능)
```

### 1.3 파일 네이밍 규칙

| 유형 | 규칙 | 예시 |
|------|------|------|
| JSP 파일 | 소문자 + 언더스코어 | `main_student.jsp`, `room_reserve.jsp` |
| Servlet | PascalCase + Servlet 접미사 | `LoginServlet.java`, `ReserveServlet.java` |
| SQL 파일 | 기능 단위 명명 | `campusnav_db.sql`, `rooms_setup.sql` |
| CSS 파일 | 소문자 | `common.css` |
| 배치 파일 | 소문자 + 언더스코어 | `compile.bat`, `compile_school.bat` |

## 2. 빌드 규칙 (Build Rules)

### 2.1 Servlet 컴파일

Servlet은 Tomcat 배포 전 **반드시 수동 컴파일**이 필요하다.

**컴파일 대상 파일:**

```
DBUtil.java
LoginServlet.java
LogoutServlet.java
GuestServlet.java
VisitorServlet.java
RegisterServlet.java
AssetServlet.java
DetailServlet.java
ReserveServlet.java
SearchServlet.java
```

**컴파일 명령어:**

```bat
javac -encoding UTF-8 ^
  -cp "%TOMCAT_HOME%\lib\servlet-api.jar;%TOMCAT_HOME%\lib\mysql-connector-j.jar" ^
  -d "%WEBAPPS%\CAN\WEB-INF\classes" ^
  "%WEBAPPS%\CAN\WEB-INF\src\com\campus\nav\*.java"
```

**자동화 스크립트:**

| 스크립트 | 대상 환경 | 설명 |
|----------|-----------|------|
| `compile.bat` | 개발 PC | 자동 Tomcat 경로 탐색, 전체 Servlet 컴파일 |
| `compile_school.bat` | 학교 PC | 고정 경로 지정, DBUtil + 인증 Servlet 컴파일 |

**컴파일 전 필수 사항:**

```
1. Tomcat 서비스 중지 (작업관리자 → 서비스 → Tomcat9 → 중지)
2. compile.bat 실행 (관리자 권한)
3. 컴파일 성공 확인 (에러 없이 종료)
4. Tomcat 서비스 재시작
```

### 2.2 JSP 파일

JSP는 Tomcat이 런타임에 자동 컴파일하므로 **별도 빌드 불필요**하다. 파일 수정 후 브라우저 새로고침으로 즉시 반영된다.

## 3. 배포 절차 (Deployment Steps)

### 3.1 DB 배포

| 단계 | 작업 | SQL 파일 | 비고 |
|------|------|----------|------|
| STEP 1 | DB·테이블 생성 | campusnav_db.sql | 반드시 최초 실행 |
| STEP 2 | 자산 데이터 적재 | campusnav_assets.sql | 8,401건, 1~3분 소요 |
| STEP 3 | 이관·폐기 데이터 적재 | campusnav_transfer_disposal.sql | 이관 140 + 폐기 2,721건 |
| STEP 4 | 교수·과목·주특기 데이터 | campusnav_add.sql | 샘플 데이터 |
| STEP 5 | 강의실 설정 | rooms_setup.sql | 64개 강의실 |

> 반드시 STEP 1 → 2 → 3 → 4 → 5 순서로 실행 (외래키 제약 위반 방지)

### 3.2 웹앱 배포

```
1. CAN 폴더 복사
   → Tomcat\webapps\ 아래에 CAN 폴더 전체 복사

2. JDBC 드라이버 설치
   → mysql-connector-j.jar → Tomcat\lib\ 폴더에 복사

3. Servlet 컴파일
   → compile.bat 실행 (Tomcat 중지 후)

4. Tomcat 재시작
   → 작업관리자 → 서비스 → Tomcat9 → 다시 시작

5. 동작 확인
   → http://localhost:8080/CAN/campuslogin.jsp 접속
   → 테스트 계정(admin/1234)으로 로그인 확인
```

### 3.3 배포 디렉토리 구조

```
Tomcat\webapps\CAN\
├── campuslogin.jsp
├── register.jsp
├── main_student.jsp / main_assistant.jsp / main_professor.jsp
├── main_admin.jsp / main_guest.jsp / main_visitor.jsp
├── search.jsp / detail.jsp / reserve.jsp / room_reserve.jsp
├── transfer.jsp / professor.jsp / navigationTest1.jsp / floorNav.jsp
├── asset_manage.jsp / disposal_admin.jsp / reservations_admin.jsp
├── getRoutes.jsp / saveRoute.jsp / saveFloorRoute.jsp
├── saveEntrancePin.jsp / saveRoomPin.jsp / deleteRoute.jsp
├── images/
│   └── logo.png / ictlayout.png (캠퍼스 평면도)
├── css/

    ├── web.xml
    ├── classes/com/campus/nav/
    │   └── *.class (컴파일된 Servlet)
    └── src/com/campus/nav/
        └── *.java (Servlet 소스)
```
