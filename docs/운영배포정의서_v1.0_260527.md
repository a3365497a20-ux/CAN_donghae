## 1. 런타임 환경 설정 (Runtime Environment Setup)

### 1.1 사전 설치 항목

| 항목 | 버전 | 설치 위치 | 확인 방법 |
|------|------|-----------|-----------|
| JDK | 8 이상 | 시스템 PATH | `java -version` |
| Apache Tomcat | 9.0.x | `C:\Program Files\apache-tomcat-9.0.115\` | 서비스 목록 확인 |
| MySQL | 8.x | 로컬 설치 | `mysql --version` |
| mysql-connector-j | 최신 | `Tomcat\lib\` | 파일 존재 확인 |

### 1.2 Tomcat 환경 설정

**server.xml — URIEncoding 설정 (한글 깨짐 방지):**

```xml
<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           URIEncoding="UTF-8" />
```

**web.xml — 서블릿 매핑:**

```xml
<servlet>
  <servlet-name>LoginServlet</servlet-name>
  <servlet-class>com.campus.nav.LoginServlet</servlet-class>
</servlet>
<servlet-mapping>
  <servlet-name>LoginServlet</servlet-name>
  <url-pattern>/login</url-pattern>
</servlet-mapping>
```

### 1.3 DB 연결 설정

DBUtil.java 내 연결 정보:

```java
private static final String URL =
    "jdbc:mysql://localhost:3306/campusnav" +
    "?useSSL=false&serverTimezone=Asia/Seoul" +
    "&characterEncoding=UTF-8&allowPublicKeyRetrieval=true";
private static final String USER = "root";
private static final String PASSWORD = "1234";
```

## 2. 시스템 시작·중지 절차

### 2.1 시스템 시작

```
1. MySQL 서비스 시작
   → 작업관리자 → 서비스 → MySQL80 → 시작

2. Tomcat 서비스 시작
   → 작업관리자 → 서비스 → Tomcat9 → 시작

3. 접속 확인
   → http://localhost:8080/CAN/campuslogin.jsp
```

### 2.2 시스템 중지

```
1. Tomcat 서비스 중지
   → 작업관리자 → 서비스 → Tomcat9 → 중지

2. MySQL 서비스 중지 (필요 시)
   → 작업관리자 → 서비스 → MySQL80 → 중지
```

### 2.3 Servlet 업데이트 절차

```
1. Tomcat 중지 (필수)
2. .java 파일 수정
3. compile.bat 실행 → .class 파일 생성
4. Tomcat 시작
5. 브라우저 접속 확인
```

## 3. 시스템 모니터링 가이드

### 3.1 정상 동작 확인 지표

| 항목 | 확인 방법 | 정상 기준 |
|------|-----------|-----------|
| 서비스 접속 | http://localhost:8080/CAN/campuslogin.jsp | 로그인 화면 표시 |
| DB 연결 | SELECT COUNT(*) FROM assets; | 8401 반환 |
| 자산 검색 | 키워드 검색 결과 표시 | 3초 이내 결과 |
| 예약 저장 | 예약 후 reservations 테이블 조회 | 레코드 INSERT 확인 |

### 3.2 Tomcat 로그 확인

| 로그 파일 | 위치 | 내용 |
|-----------|------|------|
| catalina.out / catalina.log | `Tomcat\logs\` | 서버 시작/중지, 전체 로그 |
| localhost.log | `Tomcat\logs\` | 웹앱 오류 |
| localhost_access_log | `Tomcat\logs\` | HTTP 요청 액세스 로그 |

**주요 오류 로그 패턴:**

| 오류 메시지 | 원인 | 조치 |
|-------------|------|------|
| `ClassNotFoundException: com.mysql.cj.jdbc.Driver` | JDBC JAR 없음 | mysql-connector-j.jar → Tomcat\lib\ 복사 |
| `Communications link failure` | MySQL 미실행 | MySQL 서비스 시작 |
| `error while writing .class` | Tomcat 실행 중 컴파일 | Tomcat 중지 후 재컴파일 |
| `404 The requested resource is not available` | URL 오타 또는 서블릿 미매핑 | web.xml 매핑 확인 |
| 한글 깨짐 | 인코딩 불일치 | pageEncoding="UTF-8", URIEncoding="UTF-8" 확인 |

## 4. 장애 복구 절차 (Incident Recovery)

### CASE-01 로그인 불가

| 단계 | 조치 |
|------|------|
| 1 | 테스트 계정(admin/1234)으로 재시도 |
| 2 | MySQL 서비스 실행 여부 확인 |
| 3 | LoginServlet.class 존재 여부 확인 |
| 4 | Tomcat 중지 → compile.bat 실행 → Tomcat 시작 |

### CASE-02 자산 데이터 미표시

| 단계 | 조치 |
|------|------|
| 1 | `SELECT COUNT(*) FROM assets;` 실행 → 8401 확인 |
| 2 | 0이면 campusnav_assets.sql 재실행 |
| 3 | DB 연결 정보 (localhost:3306/campusnav) 확인 |

### CASE-03 DB 연결 오류 (전체 페이지)

| 단계 | 조치 |
|------|------|
| 1 | MySQL 서비스 상태 확인 (작업관리자 → 서비스) |
| 2 | mysql-connector-j.jar → Tomcat\lib\ 존재 여부 확인 |
| 3 | DBUtil.java URL / USER / PASSWORD 확인 |
| 4 | Tomcat 재시작 |

### CASE-04 Servlet 404 오류

| 단계 | 조치 |
|------|------|
| 1 | web.xml 서블릿 매핑 확인 (/login, /logout 등) |
| 2 | WEB-INF/classes/com/campus/nav/*.class 존재 확인 |
| 3 | Tomcat 중지 → compile.bat 실행 → Tomcat 시작 |

### CASE-05 한글 깨짐

| 단계 | 조치 |
|------|------|
| 1 | JSP 상단 `pageEncoding="UTF-8"` 확인 |
| 2 | JDBC URL에 `characterEncoding=UTF-8` 포함 확인 |
| 3 | Tomcat server.xml `URIEncoding="UTF-8"` 추가 |
| 4 | Tomcat 재시작 |
