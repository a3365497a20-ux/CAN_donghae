## 4. 시스템 아키텍처

```
[클라이언트 (브라우저)]
        │  HTTP 요청
        ▼
[Apache Tomcat 9.x — WAS]
  ├── JSP 페이지 (View)
  │     campuslogin.jsp / main_*.jsp / search.jsp / detail.jsp 등
  ├── Java Servlet (Controller)
  │     LoginServlet / SearchServlet / ReserveServlet 등
  └── DBUtil.java (DB 연결 공통 유틸)
        │  JDBC (mysql-connector-j)
        ▼
[MySQL 8.x — DB 서버]
  └── campusnav 데이터베이스
        테이블: users, assets, reservations, rooms, room_reservations,
               asset_transfer, asset_disposal, professors, prof_subjects, prof_skills
```

## 5. 데이터베이스 환경

### 5.1 주요 테이블 목록

| 테이블명 | 설명 | 주요 컬럼 |
|----------|------|-----------|
| `users` | 사용자 계정 | user_id, user_pw, user_name, user_email, dept, role |
| `assets` | 자산 정보 (8,401건) | asset_no, 유형, 품목명, 제조사, 모델, 취득일, 가액, 위치 경로, 호실, 학과, 담당자, 상태, latitude, longitude, floor |
| `asset_transfer` | 자산 이관 이력 (140건) | 출발부서, 도착부서, 이관일, 이관 사유 |
| `asset_disposal` | 자산 폐기 이력 (2,721건) | 폐기일, 폐기 사유 |
| `reservations` | 자산 예약 | reserve_id, asset_no, user_id, reserve_date, start_time, end_time, status |
| `rooms` | 강의실·컴퓨터실·세미나실 (64개) | room_id, building, floor, room_no, room_name, room_type, capacity, lat, lng |
| `room_reservations` | 강의실 예약 | reserve_id, room_id, user_id, reserve_date, start_time, end_time, purpose, phone, status |
| `professors` | 교수 정보 | 교수 ID, 이름, 학과 등 |
| `prof_subjects` | 교수 담당 과목 | 과목명, 코드, 학년, 학기, 강의실, 시간 |
| `prof_skills` | 교수 주특기·연구분야 | 태그 방식 |

### 5.2 DB 구축 순서

| 순서 | SQL 파일 | 내용 |
|------|----------|------|
| STEP 1 | campusnav_db.sql | DB 생성, 테이블 8개 생성 |
| STEP 2 | campusnav_assets.sql | 자산 8,401건 INSERT |
| STEP 3 | campusnav_transfer_disposal.sql | 이관 140건 + 폐기 2,721건 INSERT |
| STEP 4 | campusnav_add.sql | 교수·과목·주특기 샘플 데이터 |

## 6. 서블릿 구성 환경

| 서블릿 클래스 | URL 매핑 | 주요 역할 |
|---------------|----------|-----------|
| LoginServlet | `/login` | 로그인 인증, 역할별 페이지 리다이렉트 |
| LogoutServlet | `/logout` | 세션 무효화 후 로그인 페이지 이동 |
| GuestServlet | `/guest` | 게스트 세션 설정 및 메인 이동 |
| VisitorServlet | `/visitor` | 외부 방문자 세션 설정 및 메인 이동 |
| RegisterServlet | `/register` | 회원가입 처리 (DB 직접 저장) |
| SearchServlet | `/search` | 자산 검색 (키워드·분류·상태 필터) |
| DetailServlet | `/detail` | 자산 상세 조회 (전체 컬럼) |
| ReserveServlet | `/reserve` | 예약 신청 및 중복 체크 |
| AssetServlet | `/asset` | 자산 관리 (수정·상태 변경) |

## 8. 확장 예정 환경

| 항목 | 내용 | 준비 상태 |
|------|------|-----------|
| Google Maps API | detail.jsp, transfer.jsp 지도 연동 | 플레이스홀더 구현 완료, API 키 미발급 |
| 실내 GPS | 1·2공학관 GPS 설치 후 assets 테이블 latitude/longitude 컬럼 연동 | DB 컬럼 사전 준비 완료 |
| QR 코드 | `/CAN/detail.jsp?id=자산번호` 로 이동하는 QR 라벨 | URL 구조 준비 완료 |
| 관리자 도착 알림 | WebSocket 또는 DB 폴링 방식 | 미개발 |
| LoginServlet DB 연동 | 현재 하드코딩 배열 방식 → DB 조회 방식으로 재컴파일 필요 | 코드 준비 완료, 미배포 |
