## 3. ERD (Entity Relationship Diagram)

```
users ──────────────────────────────────────────────────────────┐
  │ user_id (PK)                                                │
  │                                                             │
  ├──< reservations                                             │
  │       asset_no (FK → assets)                                │
  │       user_id  (FK → users)                                 │
  │                                                             │
  ├──< room_reservations                                        │
  │       room_id  (FK → rooms)                                 │
  │       user_id  (FK → users)                                 │
  │                                                             │
  └──< professors ──< prof_subjects                             │
            └──< prof_skills                                    │
                                                                │
assets ─────────────────────────────────────────────────────────┘
  │ asset_no (PK)
  ├──< asset_transfer
  ├──< asset_disposal
  └──< reservations

rooms
  │ room_id (PK)
  └──< room_reservations

entrances ──< routes
floor_routes >──(room_id)──< rooms
```

## 4. 데이터 흐름

### 4.1 자산 데이터 흐름

```
SQL 파일 (campusnav_assets.sql)
        │ 8,401건 INSERT
        ▼
    assets 테이블
        │
        ├── SearchServlet → search.jsp   (검색 조회)
        ├── DetailServlet → detail.jsp   (상세 조회)
        ├── ReserveServlet → reserve.jsp (예약 등록)
        └── AssetServlet → asset_manage.jsp (관리자 수정)
```

### 4.2 예약 데이터 흐름

```
사용자 입력 (날짜, 시간)
        │
        ▼
AJAX 중복 체크 ──── reservations / room_reservations 조회
        │
        ├── 중복: 빨간 경고 표시 → 입력 차단
        └── 가능: 초록 확인 표시 → 예약 신청 허용
                        │
                        ▼
              POST 전송 → DB INSERT → 예약완료
```

### 4.3 길찾기 데이터 흐름

```
관리자 경로 등록 (navigationTest1.jsp + 관리자 모드)
        │ saveRoute.jsp / saveFloorRoute.jsp
        ▼
  routes / floor_routes 테이블
        │ getRoutes.jsp (JSON 응답)
        ▼
사용자 경로 안내 (Canvas 위에 경로선 렌더링)
        │
        └── 출입구 도착 → floorNav.jsp (층별 실내 경로)
```

## 6. 데이터 검증 쿼리

```sql
USE campusnav;

-- 자산 건수 확인
SELECT COUNT(*) FROM assets;           -- 기대값: 8401

-- 이관 건수 확인
SELECT COUNT(*) FROM asset_transfer;   -- 기대값: 140

-- 폐기 건수 확인
SELECT COUNT(*) FROM asset_disposal;   -- 기대값: 2721

-- 강의실 건수 확인
SELECT COUNT(*) FROM rooms;            -- 기대값: 64

-- 유형별 자산 분포
SELECT asset_type, COUNT(*) as cnt
FROM assets
GROUP BY asset_type;

-- 역할별 사용자 수
SELECT role, COUNT(*) as cnt
FROM users
GROUP BY role;

-- 건물별 강의실 수
SELECT building, room_type, COUNT(*) as cnt
FROM rooms
GROUP BY building, room_type
ORDER BY building;
```
