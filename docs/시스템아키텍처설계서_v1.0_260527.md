## 1. 아키텍처 개요

본 시스템은 **3-Tier 웹 아키텍처**를 채택한다. 클라이언트(브라우저) — 애플리케이션 서버(Tomcat/JSP+Servlet) — 데이터베이스(MySQL)의 계층 구조로 구성되며, MVC 패턴에 준하여 View(JSP), Controller(Servlet), Model(DB+DBUtil)을 분리한다.

## 3. 인프라 구성 (Infrastructure)

| 구성 요소 | 기술 | 버전 | 설치 경로 |
|-----------|------|------|-----------|
| WAS | Apache Tomcat | 9.0.x | `C:\Program Files\apache-tomcat-9.0.115\` |
| DBMS | MySQL | 8.x | localhost:3306 |
| JDBC Driver | mysql-connector-j | 최신 | `Tomcat\lib\mysql-connector-j.jar` |
| JDK | Java SE | 8 이상 | 시스템 PATH 등록 |
| 웹앱 배포 경로 | — | — | `...\webapps\CAN\` |

### 빌드 구성

| 구분 | 방법 |
|------|------|
| Servlet 컴파일 | `javac -encoding UTF-8 -cp servlet-api.jar -d WEB-INF/classes *.java` |
| 자동화 스크립트 | `compile.bat` (개발 PC), `compile_school.bat` (학교 PC) |
| JSP | Tomcat이 런타임에 자동 컴파일 (재시작 불필요) |
| Servlet | 소스 수정 시 수동 재컴파일 + Tomcat 재시작 필요 |

## 4. 요청 처리 흐름 (Request Flow)

### 4.1 일반 페이지 요청 (GET)

```
Browser → GET /CAN/search.jsp?keyword=노트북
  → Tomcat: JSP 컴파일 (최초 1회)
  → JSP 내 DB 조회 (DBUtil.getConnection → PreparedStatement → ResultSet)
  → HTML 렌더링 → Browser 응답
```

### 4.2 서블릿 처리 요청 (POST)

```
Browser → POST /login (userId, userPw, selectedRole)
  → Tomcat → LoginServlet.doPost()
  → DBUtil.getConnection() → users 테이블 조회
  → 세션 생성 (loginUser, loginName, loginRole)
  → response.sendRedirect("/CAN/main_{role}.jsp")
```

### 4.3 AJAX 비동기 요청

```
Browser → fetch("GET /CAN/getRoutes.jsp?fromId=1&toId=3")
  → JSP DB 조회 → routes 테이블 → points_json 반환
  → response.setContentType("application/json")
  → JSON 응답 → Canvas 경로 렌더링
```

## 5. 보안 전략 (Security Strategy)

| 항목 | 전략 |
|------|------|
| 인증 | HttpSession 기반 서버 세션 (유효시간 30분) |
| 인가 | 모든 보호 JSP 최상단에서 `session.getAttribute("loginUser")` null 체크 |
| 역할 제어 | `session.getAttribute("loginRole")`로 역할 불일치 시 리다이렉트 |
| SQL Injection 방지 | PreparedStatement 파라미터 바인딩 전면 적용 |
| XSS 방지 | JSP 출력 시 `<%= escapeHtml(value) %>` 처리 |
| 쿠키 | 아이디 저장 쿠키만 사용, 민감정보 쿠키 미사용 |
| 비밀번호 | 현재 평문 저장 → 향후 BCrypt 해싱 적용 예정 |
| 로그아웃 | POST 방식 전용, GET 직접 접근 차단 |

## 6. 계층별 책임 요약

| 계층 | 구성 요소 | 책임 |
|------|-----------|------|
| View | JSP | 화면 렌더링, 사용자 입력 폼, HTML 생성 |
| Controller | Java Servlet | 요청 수신, 파라미터 검증, 비즈니스 로직 호출, 응답 처리 |
| Utility | DBUtil.java | DB 연결 생성/해제 공통화, 예외 처리 |
| Model | MySQL Tables | 데이터 영속성, 무결성 제약, 인덱스 |
| Static | CSS / Images | 공통 스타일, 로고, 평면도 이미지 |
