## 1. 테스트 전략

| 테스트 유형 | 방법 | 대상 |
|-------------|------|------|
| 단위 테스트 | 모듈별 독립 동작 확인 | 각 Servlet, JSP 페이지 |
| 통합 테스트 | 역할별 전체 시나리오 실행 | 로그인 → 검색 → 예약 흐름 |
| 인터페이스 테스트 | HTTP 요청/응답, AJAX 응답 검증 | API 엔드포인트 |
| UI 테스트 | PC/모바일 브라우저 레이아웃 확인 | 반응형 화면 |
| 보안 테스트 | 세션 없이 직접 URL 접근, SQL Injection 시도 | 인증/인가 모듈 |

### BDD 시나리오 형식

```
Feature: [기능명]
  Scenario: [시나리오명]
    Given  [사전 조건]
    When   [행동]
    Then   [기대 결과]
```

## 2. 단위 테스트 케이스

### UT-01 LoginServlet — 정상 로그인

```
Feature: 사용자 로그인
  Scenario: 유효한 계정으로 학부생 로그인
    Given  DB에 student1/pass1234/student 계정이 존재한다
    When   아이디 "student1", 비밀번호 "pass1234", 역할 "student" 선택 후 로그인 버튼 클릭
    Then   세션에 loginUser="student1", loginRole="student" 저장
    And    /CAN/main_student.jsp 로 리다이렉트된다
```

| 항목 | 내용 |
|------|------|
| 테스트 ID | UT-01 |
| 대상 | LoginServlet.doPost() |
| 입력 | userId=student1, userPw=pass1234, selectedRole=student |
| 기대 결과 | HTTP 302 → /CAN/main_student.jsp |
| 결과 | ☐ 통과 / ☐ 실패 |

## 3. 통합 테스트 케이스

### IT-01 학부생 전체 흐름 테스트

```
Feature: 학부생 자산 예약 전체 흐름
  Scenario: 자산 검색 후 예약 완료
    Given  student1/pass1234 계정이 존재한다
    When   로그인 → main_student.jsp 이동
    And    "노트북" 키워드로 자산 검색
    And    검색 결과에서 자산 행 클릭 → detail.jsp 이동
    And    [예약하기] 버튼 클릭 → reserve.jsp 이동
    And    날짜·시간 선택 후 AJAX 중복 체크 통과
    And    [예약 신청] 버튼 클릭
    Then   reservations 테이블에 예약 레코드 INSERT됨
    And    예약 완료 메시지 표시
```

| 항목 | 내용 |
|------|------|
| 테스트 ID | IT-01 |
| 역할 | student |
| 경로 | 로그인 → 검색 → 상세 → 예약 |
| 결과 | ☐ 통과 / ☐ 실패 |
