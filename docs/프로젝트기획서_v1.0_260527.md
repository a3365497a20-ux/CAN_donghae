## 1. 프로젝트 승인 정보 (Project Approval)

### 1.1 프로젝트 배경

ICT폴리텍대학은 경기도 광주시 순암로 16-26에 위치하며, 제1공학관·제2공학관·대학본부·학생회관·산학협력관 등 복수의 건물에 걸쳐 총 **8,401건**의 교내 자산을 보유하고 있다. 그러나 현재 자산 정보는 텍스트 경로 방식으로만 관리되며 자산 검색·예약·이관 조회를 통합 처리하는 시스템이 부재하여 업무 비효율이 발생하고 있다.

또한 재학생·교직원·외부 방문자 등 다양한 이해관계자가 캠퍼스 자원을 이용하나, 역할별로 구분된 접근 체계가 없어 정보 혼선이 빈번하다. 이에 교내 자원을 효율적으로 검색·예약·관리하고, 실내 길찾기까지 통합 제공하는 웹 기반 내비게이션 시스템의 필요성이 제기되었다.

### 1.2 프로젝트 승인 근거

| 항목 | 내용 |
|------|------|
| 요청 주체 | ICT폴리텍대학 AI소프트웨어학과 |
| 승인 유형 | 학과 팀 프로젝트 |
| 개발 기간 | 약 6.5주 |
| 접속 URL | http://localhost:8080/CAN/campuslogin.jsp |
| 최종 업데이트 | 2026-04-14 |

## 2. 비즈니스 요구사항 (BRD: Business Requirements Document)

### 2.1 핵심 비즈니스 문제

| # | 문제 | 영향 |
|---|------|------|
| 1 | 자산 위치가 텍스트 경로로만 관리되어 실물 탐색이 어려움 | 자산 활용 효율 저하 |
| 2 | 자산 검색·예약·이관 조회가 통합 시스템 없이 수작업 관리 | 업무 시간 낭비 |
| 3 | 강의실·자산 예약 시 실시간 중복 확인 불가 | 예약 충돌 발생 |
| 4 | 역할별 접근 권한 구분이 없어 정보 혼선 | 보안 취약, 정보 오용 |
| 5 | 외부 방문자와 재학생의 서비스 구분 부재 | 불필요한 정보 노출 |

### 2.2 기대 비즈니스 효과

| 대상 | 기대 효과 |
|------|-----------|
| 학부생 | 자산 번호·품목명으로 빠른 자산 탐색 및 예약 |
| 교직원 | 이관 이력·자산 현황 실시간 조회로 관리 효율 향상 |
| 관리자 | 통계 대시보드로 자산·예약·이관·폐기 현황 일원화 |
| 외부 방문자 | 계정 없이 강의실 예약 및 캠퍼스 길찾기 이용 |
| 전체 | 실내 GPS 연동 시 위치 기반 자원 탐색 가능 |

### 2.3 핵심 비즈니스 요구사항

| BR-ID | 요구사항 | 우선순위 |
|-------|---------|---------|
| BR-01 | 교내 자산 8,401건을 키워드·분류·상태로 통합 검색할 수 있어야 한다 | 상 |
| BR-02 | 자산 및 강의실 예약 시 실시간 중복 체크가 가능해야 한다 | 상 |
| BR-03 | 역할(학부생/조교/교수/관리자/게스트/방문자)별로 기능 접근이 구분되어야 한다 | 상 |
| BR-04 | 자산 이관 이력을 부서 흐름으로 조회할 수 있어야 한다 | 중 |
| BR-05 | 캠퍼스 내 출발지에서 목적지까지 경로를 안내할 수 있어야 한다 | 중 |
| BR-06 | 관리자는 전체 자산·예약·이관·폐기 현황을 통계로 확인할 수 있어야 한다 | 상 |
| BR-07 | 외부 방문자는 계정 없이 공간 예약 및 길찾기를 이용할 수 있어야 한다 | 중 |

## 3. 시스템 범위 (SS: System Scope)

### 3.1 범위 내 (In-Scope)

| 범주 | 포함 기능 |
|------|-----------|
| 사용자 인증 | 로그인, 로그아웃, 회원가입, 게스트·외부 방문자 진입 |
| 자산 관리 | 자산 검색, 상세 조회, 자산 예약, 이관 이력 조회, 폐기 이력 조회 |
| 공간 관리 | 강의실·컴퓨터실·세미나실 조회 및 예약 |
| 길찾기 | 캠퍼스 평면도 기반 출입구 간 경로 안내, 층별 실내 경로 안내 |
| 관리자 기능 | 자산 등록·수정, 예약 관리, 폐기 관리, 통계 대시보드 |
| 교수 자원 | 교수 목록, 과목 등록, 주특기·연구분야 등록 |

### 3.2 범위 외 (Out-of-Scope)

| 항목 | 사유 |
|------|------|
| Google Maps API 실시간 연동 | API 키 미발급, 향후 확장 예정 |
| 실내 GPS 위치 추적 | 하드웨어 미설치, 향후 확장 예정 |
| QR 코드 라벨 인쇄 | 별도 앱(앵그로우) 연동 필요, 향후 확장 예정 |
| 관리자 도착 알림 | WebSocket 서버 미구성, 향후 확장 예정 |
| 모바일 네이티브 앱 | 웹 브라우저 기반으로 제한 |
| 외부 인터넷 공개 배포 | 교내 로컬(localhost) 환경으로 제한 |

### 3.3 시스템 경계

```
┌─────────────────────────────────────────────────────────┐
│                   ICT CampusNav 시스템                   │
│                                                         │
│  [사용자 브라우저]                                        │
│       ↕ HTTP                                            │
│  [Apache Tomcat 9.x — JSP/Servlet]                      │
│       ↕ JDBC                                            │
│  [MySQL 8.x — campusnav DB (자산 8,401건)]               │

│  [Google Maps API]  — 향후 연동 예정 (현재 미포함)         │
│  [실내 GPS 수신기]  — 향후 연동 예정 (현재 미포함)          │
│  [QR 앵그로우 앱]   — 향후 연동 예정 (현재 미포함)          │
└─────────────────────────────────────────────────────────┘
```

## 5. 제약 사항 및 가정 (Constraints & Assumptions)

### 5.1 제약 사항

| 유형 | 내용 |
|------|------|
| 기술 | JSP/Servlet + MySQL + Tomcat 9.x 기술 스택 고정 |
| 환경 | 로컬 환경(localhost:8080) 배포, 외부 접속 불가 |
| 보안 | 비밀번호 현재 평문 저장 (향후 해싱 처리 필요) |
| 빌드 | Servlet은 javac 수동 컴파일 필요 (compile.bat 제공) |
| 데이터 | 자산 데이터 8,401건 SQL 파일로 초기 적재 |

### 5.2 가정

| # | 가정 사항 |
|---|-----------|
| 1 | 개발·운영 환경에 MySQL 8.x 및 Tomcat 9.x가 사전 설치된다 |
| 2 | mysql-connector-j.jar가 Tomcat lib/ 폴더에 배치된다 |
| 3 | 초기 자산 데이터는 SQL 파일 4개(STEP 1~4)를 순서대로 실행하여 적재한다 |
| 4 | 실내 GPS 및 Google Maps API는 향후 별도 작업으로 연동한다 |
