# I/F 설계서 (인터페이스 설계서)

**프로젝트명:** ICT 교내 자원 내비게이션 시스템 (CampusNav)
**기관:** ICT폴리텍대학 AI소프트웨어학과
**팀원:** 박승순 · 권동해 · 원태연 · 이수혁
**작성일:** 2026-05-27
**버전:** v1.0

---

## 1. 인터페이스 설계 개요

### 1.1 인터페이스 유형

본 시스템에서 발생하는 인터페이스는 아래 세 가지 유형으로 분류된다.

| 유형 | 설명 | 해당 인터페이스 |
|------|------|-----------------|
| 사용자 ↔ 시스템 | 브라우저에서 서버로 HTTP 요청 | 로그인, 검색, 예약 등 모든 화면 |
| 시스템 ↔ DB | Servlet/JSP에서 MySQL로 JDBC 통신 | DBUtil.java 경유 쿼리 |
| 시스템 ↔ 외부 API | 서버 또는 클라이언트에서 외부 서비스 호출 | Google Maps, GPS (예정), AJAX |

### 1.2 통신 방식

| 방식 | 프로토콜 | 용도 |
|------|----------|------|
| HTTP GET | HTTP/1.1 | 화면 조회, 검색, 상세 페이지 |
| HTTP POST | HTTP/1.1 | 로그인, 예약 신청, 회원가입, 로그아웃 |
| AJAX (fetch/XHR) | HTTP/1.1 비동기 | 예약 중복 체크, 경로 데이터 조회 |
| JDBC | TCP/IP | Servlet/JSP ↔ MySQL 8.x |

## 3. 인터페이스 상세 명세

### IF-01 로그인

| 항목 | 내용 |
|------|------|
| URL | `POST /login` |
| 요청 파라미터 | userId (String), userPw (String), selectedRole (String), saveId (String, optional) |
| 처리 로직 | DB users 테이블에서 userId 조회 → 비밀번호 일치 확인 → 역할 일치 확인 → 세션 생성 |
| 세션 저장 값 | loginUser (userId), loginName (user_name), loginRole (role) |
| 성공 응답 | 역할별 메인 페이지로 redirect |
| 실패 응답 | campuslogin.jsp forward (errorMsg 속성 설정) |
| 오류 케이스 | 아이디 없음 / 비밀번호 불일치 / 역할 불일치 |
| 쿠키 처리 | saveId='on' 시 savedId 쿠키 30일 저장, 미선택 시 쿠키 삭제 |
| 세션 유효시간 | 30분 (1800초) |

**역할별 리다이렉트:**

| role 값 | 이동 URL |
|---------|----------|
| student | /CAN/main_student.jsp |
| assistant | /CAN/main_assistant.jsp |
| professor | /CAN/main_professor.jsp |
| admin | /CAN/main_admin.jsp |
| visitor | /CAN/main_visitor.jsp |
| 기타/guest | /CAN/main_guest.jsp |

### IF-06 자산 검색

| 항목 | 내용 |
|------|------|
| URL | `GET /search` |
| 요청 파라미터 | keyword (String), type (String), status (String), page (int, default=1) |

## 4. JDBC 인터페이스

### 4.1 DBUtil.java 공통 연결 구조

```
[Servlet / JSP]
      │
      ▼
DBUtil.getConnection()
      │ DriverManager.getConnection(url, user, password)
      ▼
MySQL 8.x (campusnav DB)
      │
      ▼
PreparedStatement 실행 (쿼리 + 파라미터 바인딩)
      │
      ▼
ResultSet 처리 → 데이터 반환
      │
      ▼
DBUtil.close(rs, ps, conn) → 자원 해제
```

### 4.2 JDBC 연결 정보

| 항목 | 값 |
|------|----|
| JDBC URL | `jdbc:mysql://localhost:3306/campusnav?characterEncoding=UTF-8&serverTimezone=Asia/Seoul` |
| Driver | `com.mysql.cj.jdbc.Driver` |
| 사용자 | root |
| 비밀번호 | 1234 |
| Driver JAR | mysql-connector-j.jar (Tomcat lib/ 위치) |

## 6. 인터페이스 오류 처리

| 오류 상황 | 처리 방식 | 사용자 피드백 |
|-----------|-----------|---------------|
| DB 연결 실패 | try-catch → System.err 로그 출력 | "DB 연결 오류" 메시지 표시 |
| 세션 없이 보호 페이지 접근 | JSP 최상단 세션 체크 → 로그인 페이지 redirect | 로그인 페이지로 이동 |
| 존재하지 않는 자산번호 조회 | ResultSet empty → "자산 없음" 메시지 | "해당 자산을 찾을 수 없습니다" |
| 예약 중복 | SELECT로 충돌 확인 → 오류 반환 | 빨간 경고 표시 |
| 아이디 중복 (회원가입) | SELECT 중복 체크 → forward | "이미 사용 중인 아이디입니다" |
| AJAX 요청 실패 | fetch catch → setResult 오류 메시지 | "오류: [메시지]" 표시 |
| JDBC JAR 없음 | ClassNotFoundException | 전체 페이지 DB 오류 |
| 한글 깨짐 | pageEncoding=UTF-8, JDBC URL characterEncoding=UTF-8 | 정상 처리 |
